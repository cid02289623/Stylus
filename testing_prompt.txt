# Phase 6 Pilot Test README

## Purpose

This Python script is used to test the **Phase 6 startup calibration** stage of the PIC18 gesture project.

The goal of this test is **not** to prove final gesture classification accuracy.  
Instead, it is used to verify that the PIC firmware can:

1. receive framed UART gyro data,
2. learn a startup bias for `Gx` and `Gy` while the system is still,
3. subtract that bias during a no-motion capture,
4. report the final accumulated `Dx` and `Dy`,
5. estimate a suitable displacement threshold `Tdisp` from measured no-motion data.

This supports the Phase 6 objective of building a **data-based threshold**, rather than guessing one.

## What the script sends to the PIC

The script sends the same 9-byte UART frame format expected by the PIC firmware:

- byte 0 = `0xAA`
- byte 1 = `Gx low`
- byte 2 = `Gx high`
- byte 3 = `Gy low`
- byte 4 = `Gy high`
- byte 5 = `Gz low`
- byte 6 = `Gz high`
- byte 7 = button byte
- byte 8 = checksum over bytes 1..7 mod 256

The script first sends a **startup stillness stream** so the PIC can learn the bias.

It then runs repeated **no-motion button capture trials**, where the input stays close to the learned still value, so the PIC should produce only small accumulated `Dx` and `Dy`.

---

## What the PIC should send back

The Phase 6 pilot firmware sends two special UART messages:

### 1. Bias frame after startup calibration
Format:

- `0xB1`
- `bias_gx_l`
- `bias_gx_h`
- `bias_gy_l`
- `bias_gy_h`
- checksum

This lets the Python script print the learned startup bias.
